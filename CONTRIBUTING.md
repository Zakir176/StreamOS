# Contributing to StreamOS 🤝

First off, thank you for considering contributing to StreamOS! It's people like you that make StreamOS a great tool for everyone.

## 🌈 Code of Conduct

By participating in this project, you agree to abide by the terms of our Code of Conduct (standard contributor covenant).

## 🛠️ Development Workflow

1.  **Fork the repository** and create your branch from `main`.
2.  **Set up your environment** (see the [Development Guide](docs/DEVELOPMENT.md)).
3.  **Make your changes**. Ensure your code follows the existing style and patterns.
4.  **Test your changes**. Run the backend and frontend to verify everything works as expected.
5.  **Submit a Pull Request**.

## 🎨 Style Guidelines

### Backend (Python/FastAPI)
- Follow **PEP 8** conventions.
- Use type hints for function arguments and return values.
- Keep functions small and focused on a single task.
- Document complex logic with comments.

### Frontend (Vue/JavaScript)
- Use the **Composition API** for Vue components.
- Keep components modular and reusable.
- Follow the "Liquid Glass" design tokens in `assets/styles.css`.
- Use descriptive variable and function names.

## 📝 Commit Messages
- Use the imperative mood ("Add feature" instead of "Added feature").
- Keep the first line short (under 50 characters).
- Reference issues and pull requests when applicable.

## ❓ Questions?
If you have any questions, feel free to open an issue for discussion!
